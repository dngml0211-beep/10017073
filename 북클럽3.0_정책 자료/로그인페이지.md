# 북클럽 3.0 예상 로그인 프로세스

## 1. 개요 및 이슈 사항
* **이슈 지점:** 대부분의 고객이 아이디가 없음 (SRT 역시 계약 기반 서비스이므로 간편/소셜 로그인 접목 불가)
* **핵심 방향:** 아이디/PW 방식이 아닌, **휴대폰 인증 후 자동 로그인** 방식 채택

---

## 2. 로그인 정책

**🔑 Key Data:** 이름, 생년월일, 전화번호

### 2.1 로그인 프로세스
* **LEP 고객 DB 조회:** 이름/생년월일/전화번호 기준
    * **Y (CI 있음):** 로그인 완료
    * **N (CI 없음):** 신규회원 채번(무료체험) 또는 계약 찾기 진행
        * **데이터 불일치 케이스:** 이름, 생년월일, 전화번호가 일치하지 않아 LEP 데이터 변경이 필요한 경우
        * **대응 방안:** 고객센터로 안내하거나, LEP에 등록된 계약을 변경할 수 있는 I/F 개발 필요 (고객번호, 라운지 인증번호 등으로 추가 인증 후 변경해주는 형태 고려)

### 2.2 인증 및 디바이스 관리
* **자동 로그인:** 최초 1회 인증 후 자동 로그인 진행
    * 믿을 수 있는 디바이스 자동 로그인 체크 (별도 로그아웃 하지 않을 시 계속 로그인 연동)
* **디바이스 관리 기능:** (카카오톡, 쿠팡 참조)
    * 자동 로그인 + 최근 로그인 디바이스 안내 추가 (예: x200, T583 등)
    * 원격으로 언제든 자동 로그인 연결을 끊을 수 있도록 기능 구현

### 2.3 중복 로그인 정책
* **현재 접속 우선:** 지금 막 로그인한 회원이 우선권을 가짐
* **이전 접속 해제:** 이전에 로그인한 기기는 연결이 끊어지는 형태로 구현

### 2.4 예외 처리 (로그인/탈퇴 불가)
* **로그인 불가 대상:**
    * 미납 / 연체 / 해지 고객
    * 신규 회원이었으나, 무료체험 기간이 종료된 경우
* **탈퇴 불가:** 앱 내 탈퇴 불가, 고객센터로 안내

---

## 3. 회원가입 (영업용 DB, 무료체험용)

**🔑 Key Data:** 이름, 생년월일, 전화번호

### 3.1 가입 및 체험 프로세스
* **신규 고객 채번:** 이름/생년월일/전화번호 기준으로 LEP 신규 고객 번호 생성
* **가입 절차:** 약관 동의 (마케팅 동의 등) 진행
* **무료체험 시작:** 가입 완료 시 무료체험 진행 (기간: 2주 또는 한 달 예정)

### 3.2 체험 종료 후 시나리오
* **가입 전환 시:** 지속 이용 가능
* **미전환 시:** 이용 불가 (로그인 차단)
